教学邦-网校平台

网校和公开课服务网站。
简介： 教学邦是一个网校和公开课服务平台，为用户提供网校创建和分销服务平台，隶属于上海秦镜网络科技有限公司。

一、解决的问题

1.1 机构

1.1.1 培训机构

首先说下培训机构，国内的培训机构应该是五花八门，我比较熟悉的是考研机构，it技术培训，英语考试。
参加过考研的学生很大比例上都报过考研辅导班，我在学校的时候，就有n个辅导机构
it培训更多了，很多人毕业前自己或者被学校都组织参加过培训
英语培训，比较常见的四六级，托福，雅思辅导
但是从网上搜一下，没有几个机构会有一个自己单独的系统。一方面应该是需要成本；另一方面现在培训都是地方性的，拿考验说一个地方一部分人负责，报名什么的是通过登记表，课程是通过光碟。

1.1.2 学校

对于大学来说做个自己学校的平台，是一件很容易的事情，要几个计算机系的老师和同学，捣鼓一下就出来了，但是可能对很多中小学校还是需要花费一些精力和资源的。

1.1.3 企业

一个企业要发展的好，人才很重要，公司大了完全招的是经验丰富的人不太现实。
招新人，带新人，储备人才，一般公司都会做。
而每个公司都有自己的一套技术或者办事流程，所以企业的培训对企业的长期的发展很重要。
可能目前都是做点文档，弄点ppt，开个培训meeting之类的。
要单独给自己的企业做个培训系统还是需要一定的成本。

1.2 问题

单独建站成本高，维护更新难
单独建站不好积累用户，不好推广

1.3 改善

主要优势：

* 云平台：客户无需投资任何软件，硬件
* 完善培训管理功能： 组织架构，教学计划和课程体系
* 灵活的课程设计管理：模块化，序列化，过程化内容组织。支持微课化，移动化
* 先进全面教学工具： 全球独家实现手写公式，画图，在线录制视频，音频等在线听说读写画功能。
* 简便易用： 支持手机，Pad，Pc全部终端。没有前后台区分，角色权限自动控制。
* 安全，稳定，任意扩展。

主要客户对象：

企事业单位：企事业内部网络教育平台
大中专： 同时实现共享课程平台和网络教育平台
中小学： 数字校园，网络教育实验班，辅助教学
社会教学机构：独立网校或公开课

二、区别

一般一个新产品出来，总是会被问到和这个比，和那个比，到底有什么不一样的地方，核心竞争力在什么地方。
那么教学邦和那么多的mooc的区别，和新东方在线等又有什么不一样的地方?
区别应该主要是在“云平台”和“终身教育”

一个产品要做的比较好，很多人都知道应该尽量避免竞争，而是探寻合作：

  • 百度做的是搜索平台，避开了互联网时代大家激烈的竞争，反而让各个网站争相合作
  • 团购网疯起，出现了团800，做了一个团购资源的整合
  • 团购网电商火爆，出现了返利网，通过做资源整合、购物链接来从中盈利

三、云服务模式

云计算的三类服务模式：SaaS(软件即服务)、PaaS(平台即服务)、IaaS(基础架构即服务)
教学邦属于SaaS。
简单的可以把教学邦看成是一个网校的网店，跟taobao一样。
各种教学机构可以在教学邦建立一个单独的，个性化的网校:

  • 单独的主页
  • 单独的资源管理（学生，课程…）

四、实现

4.1 实现技术

现在公司初创，可以选择一些比较“高大上”的流程，比如javascript全栈式开发。
node.js + AngularJS + Bootstrap
可以很快开发出一个很酷的原型。

4.2 云平台搭建

肯定没有必要自己去搭建整个云基础，现在的云供应商已经比较多。
选择云平台前需要考虑的点：

  • 开发的技术
  • 需要用到的服务，对于教学邦，需要的一个很重要的一个服务应该是云存储，比如视频，音频，文本文件等

  • 供应商的能力：

    国内的百度，现在支持node.js, 还有一些配套的移动端支持，比如轻应用和clouda。
    国外的AWS，aws的能力强大是无庸置疑的，加上现在进入中国，不用担心访问速度或者被墙的问题，是一个很不错的选择。

4.3 功能模块

除了上面提到的“主要优势”里的一些功能，一些细节上的需求可能会有：

4.3.1 做笔记

对于学习来说，笔记很重要，因为学习的东西很多，不可能一次都记住，需要把一些重点记下来，反复学习。
笔记的功能包括：
制作，按照适合记忆的研究分几种情况，比如树形结构等
导出，可以导出成图片或者pdf
可以在移动端浏览

4.3.2 在线做题

题目类型，题目自动判断
考试（全屏，避免学生考试在网上查资料）

4.3.3 题库的导入

如何和其他网校已经有的题库整合

五、 考虑

怎么区别于其他已经存在的五花八门的平台
如何从小的点先做起来再慢慢拓展，比如先做一个社交型的题库，或者可以帮助记忆的flashcards
面向的用户
如何建立用户粘性
有没有相关人脉资源，比如学校或者相关从事教育的人
怎么样让有经验的个人也能从中获利，比如开课或者分享
如果说朝着终身学校的方向做，是不是要考虑下学校一般的虚拟场景：比如学生会，图书馆，班干，辅导员，老师，等等，这样应该能比较好的区分与现有的平台
学习一个课程，必须要每周有作业，作业可以系统自动评改或导师评改

六、幻想

6.1

如果做的好，可以和比较多的学校合作，可以建立一个软硬件结合的产品，那样甚至可能会淘汰点读机

6.2

如果做的权威了，可以成为“终生大学”，很多之前没好好学的人，可以重新学习，学习合格了，用人单位一样认可

6.3

当学习成果（比如分数）会被人认可，可能在路上的人，都会考虑用移动app学点东西

七、随记

兄弟院校，分享资源
做个论坛，建立活跃生态圈

PS

实际操作的时候还需要多做调查，多和学校，多和同学沟通，看看页面设计和功能实现是否满足他们的期望。
