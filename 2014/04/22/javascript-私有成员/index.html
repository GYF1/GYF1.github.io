JavaScript 私有成员

在JavaScript中如何实现私有变量和私有方法
翻译：原文《Private Members in JavaScript》
在翻译的基础上加了一些便于理解的例子。

因为在js里面，object不能定义私有的变量和方法，有些人会因此就认为js里不支持定义私有变量和方法。这是一种误解。js里可以使用私有成员。下面介绍如何实现。

对象（Objects）

js里的一切都是对象，可以把任何东西都看成对象，数组是对象，函数是对象，对象还是对象。那么，什么是对象呢？对象是键值对（name-value pairs）的集合。键都是string的，但是值可以是string，number，boolean，数组，函数和对象。对象通常用来实现一个hash表，这样我们就能根据key很快的去找到对应的value。

如果对象的其中一个值是函数，也可以说成方法。当这个对象的方法被调用的时候，“this”变量就会加入到对象中。这个方法就能通过“this”来访问对象实例中的所有变量。
举个例子：

var objV1=4;
var objV2=5;
var objV3=6;

var obj = {
    objV1:1,
    objV2:2,
    objV3:3,
    objMethod: function(){
        console.log('v1::'+this.objV1);
        console.log('v2::'+this.objV2);
        console.log('v3::'+this.objV3);
    } 
}

obj.objMethod();

对象也可以在构造器中定义，也就是说通过函数来初始化对象。构造器提供的功能就类似于其他面向对象语言中“类”提供的功能，包括静态变量和静态方法。
举个例子：

function Person(n, age){
    this.id = 1;
    this.country = 'cn';
    this.name = n;
    this.age = age;
    this.sayName = function(){
        console.log('name is::'+this.name);
    };
}
var person1 = new Person('Tom', 10);
var person2 = new Person('Jack', 11);
person1.sayName();
console.log('person1 id:1:'+person1.id);
person2.id = 2;
console.log('person1 id:2:'+person1.id);
console.log('person2 id::'+person2.id);

公有的（Public）

一个对象的所有成员默认都是public的，所有的函数都能访问，更新，删除对象成员，或者增加成员。有两种方法可以在一个新的对象中增加成员

在构造器里添加

In the constructor
这种方法是用来初始化一些实例变量的，构造器的“this”变量用来给对象增加成员，例如

function Container(param) {
    this.member = param;
}

那么，如果我们初始化一个实例的时候

var myContainer = new Container('abc');

myContainer的成员就有“abc”了

在原型里添加

In the prototype
这种方法主要是用来添加一些公共方法的。当我们对某一个对象查找一个成员时，首先从对象里面找，如果对象里面没有，那么会从这个对象的原型里面去找（在一个对象里面会有一个指针指向对象的原型）。原型机制可以用来实现继承。这种方法是消耗内存的。如果给一个构造器的原型添加一个方法，这个方法就会添加都所有的对象实例里面。
例如：

Container.prototype.stamp = function (string) {
    return this.member + string;
}

那么，我们就能在已经创建的实例里面使用方法

myContainer.stamp('def')

结果是 ‘abcdef’.

私有的（Private）

私有成员是通过构造器产生的，普通通过var来定义的变量和构造器的参数，都是私有变量：

function Container(param) {
    this.member = param;
    var secret = 3;
    var that = this;
}

通过上面的代码，定义了三个私有的变量：param，secret，和that， 他们加到了这个函数里面，但是外部不能访问。但是他们能够被构造器里面的的私有方法访问。例如

function Container(param) {

    function dec() {
        console.log('current::'+secret);
        if (secret > 0) {
            secret -= 1;
            return true;
        } else {
            return false;
        }
    }

    this.member = param;
    var secret = 3;
    var that = this;
}

私有方法dec会检查secret的值，如果值大于0，那么把secret减去1，返回true。
否则，返回false。
这种方法可以限制这个对象的实例的只能被使用用3次，例子可以看下一小节的代码

为了方便，我们定义了一个私有变量“that”。这是为了能够让私有的方法能够访问到对象。
私有方法不能被public方法使用，为了让私有的方法可以被使用，我们需要“”

特权的（Privileged）

一个特权的方法可以访问到私有的变量和私有方法，并且这个方法可以被公有方法和外部方法访问。我们可以删除或者替换一个特权的方法，或者强制让特权方法放弃密码。

保密方法可以被创建通过在构造器里使用“this”，例如

function Container(param) {

    function dec() {
        if (secret > 0) {
            secret -= 1;
            return true;
        } else {
            return false;
        }
    }

    this.member = param;
    var secret = 3;
    var that = this;

    this.service = function () {
        return dec() ? that.member : null;
    };
}
var myContainer = new Container('abc');
console.log('1::'+myContainer.service());
console.log('2::'+myContainer.service());
console.log('3::'+myContainer.service());
console.log('4::'+myContainer.service());

service就是一个特权的方法，通过调用myContainer.service()就会返回“abc”。当我们调用三次后，就会null。service调用了私有的方法dec，在dec里面又再次访问了私有的变量secret，service可以被其他的object或者方法访问，但是它不允许直接方法私有成员。

闭包（Closures）

之所以可能通过这种方式在 JavaScript 种实现公有，私有，特权变量正是因为闭包，闭包是指在 JavaScript 中，内部函数总是可以访问其所在的外部函数中声明的参数和变量，即使在其外部函数被返回（寿命终结）了之后。
这一点是js语言非常强大的一个特点。
私有的和特权的成员，只能在第一次构造的时候被创建，但是公有的成员可以在任何时候被添加。

模式（Patterns）

Public

function Constructor(...) {

    this.membername = value;

}
Constructor.prototype.membername = value;

Private

function Constructor(...) {

    var that = this;
    var membername = value;

    function membername(...) {...}

}

注意: 定义函数的那一句

    function membername(...) {...}

是下面这句的简写

    var membername = function membername(...) {...};

Privileged

function Constructor(...) {

    this.membername = function (...) {...};

}
