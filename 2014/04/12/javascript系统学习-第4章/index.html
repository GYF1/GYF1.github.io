JavaScript系统学习 第4章 变量、作用域和内存问题

变量其实没有那么简单。
我们在开发中需要考虑的问题：

  • 传值还是传址
  • 垃圾回收怎么做比较好
  • 基本数据类型和复杂数据类型的区别是什么

一、变量类型

在js里面只有两种变量类型，基本类型和引用类型。
这里要和数据类型分开。

1.1 动态属性

只有引用类型的变量可以增加属性

a.name = 'test';

如果是基本类型，虽然不报错，但是并不会增加新的属性

1.2 变量复制

对于基本类型是复制的变量本身, 原变量和新复制的变量，是两个完全独立的变量。
下面的代码一看就明白：

var a = 3;
var b = a;
b++;
alert(a);

对于引用类型的变量，复制的不是变量本身，而是复制的变量的引用，也就是说，新复制的变量改变了，原来的旧变量也会一直变。因为两个变量都指向同一个堆内存里面的对象。
测试的代码：

var person = new Object();
person.name = 'a';
var personClone = person;
personClone.name = 'clonea';
alert(person.name);

1.3 传参数

当我们调用函数的时候，到底是传址还是传值，是一件有讲究的事情。
在js里面都是传值调用。
但是你可能直接看起来会有点困惑，请见下面的代码。
代码1

function code1(num){
    num++;
    return num;
}
var v = 10;
var result = code1(v);
console.log(v);//10
console.log(result);//11

代码2 

function setName(person){
    person.name = 'Jack';
}
var jack = new Object();
jack.name = 'test';
setName(jack);
console.log(jack.name);//Jack

代码3

function setName(person){
    person.name = 'Jack';
    person = new Object();
    person.name = 'Lily';
}
var jack = new Object();
jack.name = 'test';
setName(jack);
console.log(jack.name);//Jack

代码1，定义的变量没有受影响，说明是传值
代码2，定义的变量被函数改变了，好像是传址调用
代码3，定义的变量是变了，但是没有变成’Lily’,说明是传值

综上所述，传参数都是传值调用。

1.4 typeof 和 instanceof 的区别

利用typeof可以区分变量是不是四种类型：字符串、数值、布尔值、还是undefined
需要注意的是，如果判断null，会返回object。
使用语法：

var v = 'test';
console.log(typeof v);//string

也就是说typeof只对基本数据类型的四种会有作用。

可是如果想判断一个变量是那种类型的对象，我们需要用到instanceof
使用语法：

result ＝ variable instanceof constructor;

二、执行环境和作用域

2.1 执行环境（execution context）

执行环境是制对于每个变量或者函数所在的区域。
每个区域都代表一个环境，比如：

  • 全局的变量有一个执行环境
  • 每个函数中的变量都有一个单独的执行环境

对于每个执行环境，js里有一个与之相关联的变量对象(variable object)。
变量对象的作用是决定某个局部变量或者函数内部可以访问到的数据。

每个函数都有自己的执行环境，当执行流进入一个函数时，函数的环境会被推入到一个环境栈中。在函数执行结束后，栈将其环境弹出，把控制权返回给之前的执行环境。函数的环境也就此被销毁，保存在其中的所有变量和函数定义也随之销毁。

2.2 作用域

当代码在一个环境中执行时，会创建变量对象的一个作用域链（scope chain）。
简单的说，作用链域的作用是为了确定执行环境中，哪些变量和函数可以访问，哪些不能访问。
内部环境可以通过作用链域访问所有的外部环境，但外部环境不能访问内部环境中的任何变量和函数。
每个环境都可以向上搜索作用链域，以查询变量和函数名；但任何环境都不能通过向下搜索作用链域而进入另一个执行环境。

全局变量：当前页面内有效
局部变量：方法内有效

Q: 运行的结果？

function hi(){
    var a;
    alert(a);
}

A: undefined

Q: 运行的结果？

function hi(){
    var a = null;
    alert(a);
}

A: null

Q: Discuss scoping in JavaScript.

JavaScript has lexical scoping based on functions but not blocks. Therefore:

//global scope
(function() {
    //anonymous function scope
    var foo = 1;
    function bar() {
        //bar function scope
        var foo = 2;
    }
    bar();
    console.log(foo); //outputs 1
    if(true) {
        var foo = 3; //redeclares foo
    }
    console.log(foo); //outputs 3
})();
