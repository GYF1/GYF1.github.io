JavaScript系统学习 第3章 基本概念

像学英语一样，学习javascript也需要学习单词和语法。
我个人觉得需要仔细学习的是使用频率在初学者种用的不是很多操作符，比作位操作符，逗号操作符等，需要考虑这些操作符在开发中有什么帮助。
关于语句，不要只熟悉很普通的，有些用的也不是很多，比如for-in语句, label语句，with语句等

一、语法

1.1 区分大小写

1.2 标示符

最佳实践：驼峰大小写格式，第一个字母小写，剩下的每个单词首字母大写

1.3 注释

目的是提高可读性

1.4 严格模式

javasript是一种解释型的语言，所以很多错误只有只运行的时候才会提示。
比如少了引号，少了括号这种很简单的错误。
所以如果也可以像编译型的语言，能够在保存后就能提示我们错误，对开发帮助会很大。
在ECMAScript 5引入了严格模式(strict mode)，可以帮我们解决这个问题。
使用方式有两种：

  • 在顶部添加如下代码：

      "use strict";

  • 在函数内部的上方包含

      function doSomething(){
      "use strict";
      //函数体
  }

“use strict”看起来像是字符串，而且也没有赋值给任何变量，启示它是一个编译指示（pragma）
不过直接加上上面的代码，帮助不是很大，我们还需要插件配合使用，我知道比较好用的是JSHint
还有一种叫做JSLint

1.5 语句

1.5.1 分号

要把加上分号当作一种习惯，因为：

  • 可读性强
  • 避免错误，例如不完整的输入
  • 没有分号，会导致代码压缩的时候报错
  • 省略分号，会由解释器确定语句的结尾。加上分号后，解释器就不必再花实践推测应该在哪里插入分号了。

1.5.2 花括号

最佳实践：始终在控制语句中使用代码块（代码块以花括号开头，以花括号结尾）。
使用了花括号，可以让编码意图更加清晰，而且也能降低修改代码时出错的机率。

二、关键字和保留字

只要记住不要使用关键字和保留字作为标示符就可以，避开。
如果遇到错误”Identifier Expected”，考虑下是不是使用了关键字或保留字。

三、变量

3.1 使用

松散类型或类型，定义使用关键字var

3.2 变量作用域

用var定义的变量会有作用域。
比如在函数里定义的变量，函数运行结束后，变量即被销毁。
入股省略了var，创建的就是一个全局变量，即使在函数中定义的，函数外部也能访问。
不过不推荐省略var，这样不好维护代码，容易造成混乱。

Q: == 和 === 的却别是什么？你会用哪个？

The equality (==) operator will compare for equality after doing necessary type casting, the identity operator (===) doesn’t do any conversions.
‘’ == ‘0’ // false
0 == ‘’ // true
0 == ‘0’ // true

false == ‘false’ // false
false == ‘0’ // true

false == undefined // false
false == null // false
null == undefined // true

Q: What is Strict Mode in JavaScript

Strict Mode has been introduced as part of ECMAScript 5 and introduces new, restricted variant of JavaScript which has following aims:

Throws errors for actions that are rather silly but previously didn’t throw an error
Throws errors for potentially unsafe actions
Disables functions that are poorly thought out
Potentially code in strict mode could run faster by eliminating mistakes that would make it difficult for JavaScript engines to perform optimizations
Strict mode can be enabled for the entire source file or on per function basis by adding a string literal “use strict” on top of the file/function i.e.

function foo(){
  "use strict";
  // ... your code ...
}
